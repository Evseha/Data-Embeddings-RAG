# RAG_TERMINAL

Веб-приложение для работы с документами на основе **Retrieval-Augmented Generation (RAG)** с гибридным поиском и защитой от галлюцинаций.

---

## Возможности

- **Загрузка документов** — загрузите `.txt`-файл и система автоматически разобьёт его на чанки и создаст векторные эмбеддинги
- **Гибридный поиск** — объединяет семантический поиск (embedding) и ключевой поиск (BM25) для точных результатов
- **Панель "Логика работы RAG"** — показывает в реальном времени ключевые слова, найденные чанки embedding-поиска, найденные чанки BM25-поиска и финальный ответ AI
- **Защита от галлюцинаций** — если максимальная схожесть чанков < 0.40, AI не вызывается, пользователь получает стандартный ответ «не найдено»
- **Стриминг ответа** — ответ AI отображается в реальном времени через SSE

## Стек технологий

| Слой | Технология |
|---|---|
| Frontend | React 19, Vite, Tailwind CSS, shadcn/ui |
| Backend | Express 5, Node.js 24, TypeScript |
| База данных | PostgreSQL + Drizzle ORM |
| Эмбеддинги | `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` (ONNX, бесплатно, без API) |
| LLM | OpenRouter free models (`nvidia/nemotron-nano-9b-v2:free` и др.) |
| Поиск | Cosine similarity (embedding) + BM25 (ключевые слова) |
| Монорепозиторий | pnpm workspaces |

## Логика гибридного поиска

```
Вопрос пользователя
       │
       ├─── Embedding-поиск ──── cosine similarity ──── TOP-5 чанков
       │         (вес 0.7)
       │
       ├─── Извлечение ключевых слов (без LLM)
       │         │
       │         └─── BM25-поиск ──── TOP-5 чанков
       │                   (вес 0.3, нормализован)
       │
       └─── Объединение и дедупликация → TOP-5 merged чанков
                    │
            max_similarity < 0.40?
                    │
            Да ─── "Информация не найдена" (LLM не вызывается)
                    │
            Нет ── Запрос к OpenRouter LLM → Стриминг ответа
```

## Быстрый старт

### Требования
- Node.js 24+
- pnpm
- PostgreSQL

### Переменные окружения

```env
DATABASE_URL=postgresql://user:password@host:5432/dbname
OPENROUTER_API_KEY=your_openrouter_key
SESSION_SECRET=your_session_secret
```

### Установка и запуск

```bash
# Установка зависимостей
pnpm install

# Применение схемы БД
pnpm --filter @workspace/db run push

# Запуск API-сервера (порт 8080)
pnpm --filter @workspace/api-server run dev

# Запуск frontend (порт авто)
pnpm --filter @workspace/rag-app run dev
```

## Защита от галлюцинаций

Двухуровневая система:

1. **Порог релевантности (уровень API):** если косинусная схожесть лучшего чанка < 0.40 — LLM не вызывается вообще. Пользователь сразу видит: *«К сожалению, я не нашёл информацию по этому вопросу в базе знаний.»*

2. **Инструкция в промпте (уровень LLM):** модели явно запрещено использовать знания вне контекста. Если контекст недостаточен — она должна ответить стандартной фразой.
